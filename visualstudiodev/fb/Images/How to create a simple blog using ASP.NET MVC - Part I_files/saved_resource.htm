                                          Vijaya Anand Mod Vijaya Anand • 2 months ago

                                          I can able to successfully solve the issue by the following query. I do some more testing and update the article and Git code.

                                          public IList<post> Posts(int pageNo, int pageSize)
                                              {
                                                 var allPosts = _session.Query<post>()
                                                                                     .Where(p => p.Published)
                                                                                     .OrderByDescending(p => p.PostedOn)
                                                                                     .Skip(pageNo * pageSize)
                                                                                     .Take(pageSize)
                                                                                     .ToList();

                                                 var postIds = allPosts.Select(p => p.Id).ToList();

                                                 return _session.Query<post>()
                                                                          .OrderByDescending(p => p.PostedOn)
                                                                          .Where(p => postIds.Contains(p.Id))
                                                                          .Fetch(p => p.Category)
                                                                          .FetchMany(p => p.Tags)
                                                                          .ToList();
                                          }

                                                                                                                                                  Vijaya Anand Mod Aman • 5 months ago

                                                                                                                                                  You don't need separate mapping classes. You've do that mapping in the derived db context (let say BlogDbContext) class as show in this article http://www.dotnetcurry.com/sho....

                                                                                                                                                  The EF repository implementation (let say EFBlogRepository implements IBlogRepository) needs BlogDbContext in it's constructor. For injecting BlogDbContext, we've to set it's bindings and we can do that in the RepositoryModule class.

                                                                                                                                                  Actually RepositoryModule class is not at all required if you are going to do all the binding work in the JustBlog MVC project but in that case the project needs dependency with EntityFramework dll which may not be a good idea. At the time of writing this... I'm working on implementing the EFBlogRepository class (atleast some methods) and hopefully I'll complete by today and share you.
